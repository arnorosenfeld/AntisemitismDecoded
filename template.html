Game Settings

Added to score. 0 stars = auto-loss.

Budget System (Abstract Units)

The budget system uses abstract units (not dollars). Income each round = base income + (fundraising stat × multiplier). Operating costs are deducted automatically.

Stat Momentum

When a stat trends in the same direction for consecutive rounds, momentum kicks in — bonuses for rising stats, penalties for falling ones.

Advisor System

Controls how many advisors the player selects from the pool at game start, and how multiple advisor bonuses combine when they recommend the same choice.

How many advisors the player picks (default: 3)
"Best" = only strongest advisor bonus applies. "Stack" = all bonuses add together.

Priority Slider Limits

These control the priority allocation sliders on the budget screen. Total is the pool to distribute; Min/Max are per-stat limits. All stats × Min must be less than Total.

Mission Star System

Mission stars replace the old mission stat bar. Players start with some stars and gain/lose them based on whether their choices are mission-aligned. Stars display in the HUD.

Win / Lose / Promotion Thresholds

Mid-round failure triggers if any single stat bar drops to or below the bar threshold. Year-end: score below the score threshold = loss; above promotion threshold = promotion (or retirement if already national).

Any single stat at or below this triggers immediate game over mid-round (default: 20)
Year-end avg below this = loss screen
Year-end avg at or above this = promotion

Round Names — one per round

Each round has four fields: Year label (e.g. "Year 1"), Round label (e.g. "Round 3"), Hebrew month, English month/description. Must match total rounds count.

Score Weights — must add up to 1.0

Character traits affect starting stats and unlock special scenario choices. Players select a configurable number at the start.

Emoji Avatar List

One emoji per line. Players choose from this list during character creation.

The four local organizations players can lead in Year 1. Each has its own culture, stat modifiers, allowed missions, political tilt, and promotion eligibility rules.

National organizations players can be promoted to. Their political tilt must match a local org's promotionEligible list. These orgs cannot be edited in terms of promotion eligibility — they are the endpoint.

Missions define the player's organizational purpose. Each choice in every scenario is tagged as mission-aligned or not, based on these IDs. The availableTo list controls which org types can select each mission — include both local and national org IDs.

Actions are what players spend action points on each round. Every action now guarantees a scenario — base effects apply first, then the scenario card loads.

Scenarios are the main event cards triggered by player actions. Each choice has three-tier mission alignment (aligned/opposed/neutral), and scenarios target political categories — the mission IDs for which that choice counts as aligned. If the list is empty, the choice is never mission-aligned.

Inbox scenarios appear as emails or messages during gameplay — styled differently from main event cards. Two are randomly queued per round and delivered after the first action and again later in the round. Like regular scenarios, each choice needs a missionAligned list.

Constituency Segments

The six demographic segments that make up the Community Trust composite. Based on Pew Research Center 2020 and JVRC 2025 data. "Conservative" here refers to the Conservative Jewish denomination, not political conservatism.

Scoring Thresholds

These determine the grade, heading, and verdict text shown at year-end based on the player's final weighted score. Also drives the Another Year / Promotion / Loss outcomes.

Stat Configuration

The four meters tracked throughout the game. Note: bar colors are now computed dynamically based on value (red → orange → blue → green → gold) and are not configurable per-stat.

Configure how political position and clout affect gameplay. The system uses a 0–100 spectrum (0=far left, 100=far right) with an optimal center-left position at 40, asymmetric penalties, and a bipartisan trap that limits centrist influence.

Political Spectrum

Political position runs 0 (far left) to 100 (far right). The optimal position is 40 (center-left), reflecting the center-left skew of American Jewish leadership. Leaning right is penalized more than leaning left by the same distance. The "bipartisan trap" penalizes centrists (position 35–65) who have low clout (<50) — they have access everywhere but influence nowhere.
Center of the "sweet spot" (default: 40)
Distance from optimal that's still effective
How much harder right-leaning is vs left (default: 1.3x)
Centrists grow clout at this rate (default: 0.6 = 60%)

Organization Starting Positions

Each organization starts with a political position and clout level. Edit these in the organization panels.

Configure the advisor pool — characters who provide counsel on high-stakes scenarios. Players select a subset of these at game start. Each advisor brings stat modifiers, political leanings, and expertise.

Headline templates for the Jewish Gazette newspaper shown between rounds. Headlines trigger based on player choices, stat thresholds, or randomly.

Breaking news events that can interrupt mid-round. These have a probability of firing after any action and use urgent red styling.

Coalition offers that appear between rounds. Players can accept up to 3 simultaneously. Each gives ongoing stat benefits but imposes constraints.

Branching conversations with characters and advisors. These fire before or after scenarios, at round start, or standalone. Each conversation is a node tree where player responses affect disposition, leading to different outcomes.
